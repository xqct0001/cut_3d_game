# Comfort Cues 商业落地与发布计划

本文面向真实发布与运营落地。Comfort Cues 的定位应保持为“桌面舒适性辅助覆盖层”和“可访问性风格 companion app”，避免宣称治疗、预防、诊断或保证缓解晕动症、眩晕、焦虑等医疗效果。

## 产品定位

Comfort Cues 是 Windows 桌面应用，为窗口化或无边框窗口化 3D 游戏提供轻量边缘视觉提示。它不注入游戏进程、不读取或写入游戏内存、不修改渲染管线、不自动输入、不提供竞技优势。

建议对外表述：

- 适合想要额外视觉参考点的玩家、内容创作者、可访问性工具用户和容易对快速镜头移动不适的人群。
- 作为个人舒适性设置补充使用，不替代游戏内 FOV、镜头晃动、动态模糊、帧率、鼠标灵敏度等设置。
- 效果因人、游戏、显示设备和游玩环境而异，不承诺适用于所有游戏或所有用户。

避免对外表述：

- “治疗晕动症”“医学级缓解”“保证不晕”“适合所有 FPS”“不会被任何反作弊拦截”。
- “提升竞技表现”“增强瞄准”“绕过游戏限制”。

## 目标用户与用例

优先用户：

- Windows PC 玩家，主要游玩第一人称或第三人称 3D 游戏。
- 需要轻量视觉参考提示、但不想安装注入式插件或游戏 Mod 的用户。
- 游戏开发者、测试人员、主播，用于演示舒适性辅助方案。

首批用例：

- 窗口化或无边框窗口化游戏中，根据鼠标或手柄视角变化显示边缘提示。
- 为不同游戏保存 profile，并通过 exe 名或窗口标题自动匹配。
- 用调试/校准模式确认覆盖层是否工作，再切回低可见度日常模式。

## 发布渠道路线

### 官网与 itch.io 路线

这是最适合早期验证的路线，门槛低、迭代快，适合先发布 portable zip。

交付物：

- `ComfortCues-<version>-windows-x64.zip`
- SHA-256 校验和
- 中文与英文发布说明
- 隐私说明、免责声明、兼容性说明、反作弊说明
- 一页式官网或 itch 页面：产品定位、截图/GIF、支持范围、下载入口、常见问题

定价建议：

- 早期可采用免费试用 + 自愿付费，或 itch “pay what you want”。
- 若转为付费，建议低价一次性买断，并提供清晰退款说明。

上线检查：

- portable zip 解压后可运行，不要求管理员权限。
- 首次运行与托盘驻留行为写清楚。
- 明确仅支持 Windows x64、窗口化/无边框窗口化，不支持独占全屏。
- 明确用户配置保存在 `%APPDATA%\Comfort Cues\`。

### Steam 路线

Steam 适合在产品稳定、素材完整、退款与支持流程成型后进入。Steam 用户对兼容性和反作弊风险非常敏感，商店页必须保守、透明。

推荐分类：

- Software / Utilities
- Accessibility 或 Game Development 相关标签可视审核情况使用，不应把产品描述成游戏外挂、Mod 或性能增强工具。

交付形态：

- 优先提供安装型构建或稳定 portable runtime，由 Steam 管理安装目录和更新。
- 保留独立日志与配置目录，不把用户 profile 写入 Steam 安装目录。
- 支持 Steam 云同步前需谨慎评估 profile 是否包含窗口标题等本地信息；默认不建议同步。

商店页重点：

- 清楚说明“不注入、不读写游戏内存、不自动操作游戏”。
- 清楚说明“部分游戏、反作弊或覆盖层冲突场景可能不可用”。
- 展示窗口化/无边框窗口化运行方式。
- 使用普通舒适性、可访问性措辞，不使用医疗疗效承诺。

发布前门槛：

- 至少完成 Windows 10/11 主流分辨率和多显示器基础测试。
- 准备 Steam 支持页面：启动失败、覆盖层不可见、被游戏遮挡、反作弊提示、杀软误报、如何退出托盘。
- 准备退款政策说明：若用户主要游戏无法兼容，应按 Steam 规则处理退款。

### Microsoft Store 路线

Microsoft Store 更适合后续进入，因为打包、权限、签名、更新和商店政策要求更严格。当前 portable zip 形态不等同于 Store 交付形态。

可能路线：

- MSIX 打包，签名后上架。
- 若覆盖层、托盘、全局输入读取或窗口枚举能力受 Store 沙盒限制，需要评估是否使用 unpackaged Win32 应用分发方式。

重点风险：

- Store 政策对后台运行、输入监听、隐私披露和用户同意更严格。
- 若实现依赖全局鼠标/手柄输入、顶层透明窗口、托盘常驻，需要逐项确认权限与审核解释。
- Store 页面对健康相关表述更敏感，应使用“comfort overlay”“visual reference cues”等中性措辞。

进入条件：

- 完成隐私政策与数据流审计。
- 完成签名、安装、卸载、自动更新、日志目录清理验证。
- 明确最小系统版本、架构、依赖项和受限功能。

## 商业模式

建议分阶段：

- 验证期：官网/itch 免费或自愿付费，收集兼容性反馈。
- 稳定期：低价买断，包含基础 profile、手动绑定、调试校准、常规更新。
- 专业期：可考虑团队授权、开发者演示授权或高级 profile 管理，但不应把安全或舒适性基础能力拆成付费墙。

不建议：

- 订阅制作为首发模式。
- 对特定热门竞技游戏做强绑定营销。
- 销售“保证通过反作弊”或“竞技增强”承诺。

## 合规与信任材料

必须准备：

- 隐私说明：本地保存哪些数据、是否联网、是否上传窗口标题或进程名。
- 免责声明：不是医疗设备，不提供医学建议，不保证减少不适；如出现不适应停止使用并咨询专业人士。
- 反作弊说明：不注入、不改内存、不自动输入；但无法保证所有游戏、平台或反作弊都允许外部覆盖层。
- 兼容性说明：仅 Windows x64，优先窗口化/无边框窗口化；独占全屏通常不可用。
- 安全说明：下载来源、校验和、签名状态、误报处理方式。

建议准备：

- 已测试游戏与环境矩阵。
- 已知不兼容列表。
- 日志收集模板，默认要求用户主动提供，不自动上传。
- 变更日志和回滚下载链接。

## 发布节奏

### v1.0 公开验证

目标是确认真实用户能下载、启动、绑定窗口、看见覆盖层、退出程序，并理解限制。

发布材料：

- 官网/itch 下载页
- 中文/英文 release notes
- `docs/troubleshooting.md`
- `docs/profile-schema.md`
- SHA-256 校验和

验收指标：

- 首次启动失败率可追踪。
- 主要问题能归类到运行库、覆盖层不可见、独占全屏、反作弊/杀软、profile 匹配。
- 文案没有医疗疗效夸大。

### v1.1 稳定化

目标是降低支持成本。

建议投入：

- 内置诊断页：当前窗口、匹配 profile、覆盖层状态、输入来源、日志路径。
- 更清晰的 profile 编辑和重置流程。
- 多显示器和 DPI 场景测试。
- 常见反作弊游戏的保守兼容说明。

### v1.2 渠道化

目标是为 Steam 和 Store 准备。

建议投入：

- 安装器或 MSIX 评估。
- 代码签名。
- 商店素材：图标、截图、短视频、胶囊图。
- 隐私政策网页和支持邮箱。
- 自动更新或渠道更新策略。
