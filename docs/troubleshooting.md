# Comfort Cues 故障排查与兼容性说明

本文用于用户支持、发布页 FAQ 和内部排障。所有说明都应保持保守：Comfort Cues 是外部桌面覆盖层，不保证适用于所有游戏、反作弊、录屏工具或显示设置。

## 快速检查

遇到问题时先确认：

1. 操作系统是 Windows x64。
2. 游戏运行在窗口化或无边框窗口化模式，不是独占全屏。
3. `ComfortCues.exe` 正在运行；如果主窗口关闭，检查系统托盘。
4. 已在 Comfort Cues 中绑定当前窗口，或 profile 的 `match_exe` / `match_title` 能匹配目标游戏。
5. 调试/校准模式开启后，边缘提示是否变得明显。
6. 游戏、录屏软件、显卡覆盖层和安全软件是否拦截了桌面覆盖层。

## 覆盖层不可见

可能原因：

- 游戏处于独占全屏模式。
- 游戏或平台覆盖层始终置顶，遮住 Comfort Cues。
- 当前窗口没有匹配到 profile。
- `max_opacity` 太低，或未开启调试/校准模式。
- 多显示器、DPI 缩放或 HDR 设置导致覆盖层位置异常。

处理步骤：

1. 将游戏切换为窗口化或无边框窗口化。
2. 在 Comfort Cues 中点击“选择窗口”，选中游戏窗口后点击“绑定选中”。
3. 临时开启调试/校准模式，确认提示是否可见。
4. 暂时关闭其他覆盖层：Steam Overlay、Discord Overlay、Xbox Game Bar、NVIDIA/AMD Overlay、录屏软件叠加层。
5. 如果使用多显示器，先把游戏和 Comfort Cues 放在主显示器测试。
6. 重启 Comfort Cues 和游戏后再测试。

## 输入没有响应

可能原因：

- profile 中关闭了 `enable_mouse` 或 `enable_gamepad`。
- 游戏窗口未获得焦点。
- 手柄由 Steam Input、厂商驱动或游戏独占处理，桌面层无法稳定读取。
- `deadzone` 过大，小幅移动被过滤。

处理步骤：

1. 确认目标 profile 中 `enable_mouse = true` 或 `enable_gamepad = true`。
2. 点击游戏窗口，让游戏获得焦点。
3. 临时降低 `deadzone`，例如从 `0.080` 调到 `0.050`。
4. 分别测试鼠标与手柄，确认是哪一种输入来源失效。
5. 对手柄问题，先关闭 Steam Input 或更换为游戏原生手柄输入进行对照测试。

## 反作弊与竞技游戏

Comfort Cues 的设计原则：

- 不注入游戏进程。
- 不 Hook 游戏渲染管线。
- 不读取或写入游戏内存。
- 不发送合成输入，不自动瞄准，不自动操作。
- 只作为桌面覆盖层显示轻量视觉提示。

仍需注意：

- 反作弊系统和游戏平台规则会变化，外部覆盖层不一定被所有游戏允许。
- 某些竞技游戏可能限制任何第三方覆盖层，即使该覆盖层不修改游戏。
- 如果游戏、平台或赛事规则禁止第三方覆盖层，应遵守其规则并关闭 Comfort Cues。
- 发布材料不得承诺“保证不封号”或“保证通过反作弊”。

建议用户操作：

1. 先在非排位、离线、训练场或私人房间测试。
2. 查看游戏官方对第三方覆盖层、辅助工具和可访问性软件的说明。
3. 出现反作弊提示、游戏崩溃或账号风险警告时，立即退出 Comfort Cues。

## 隐私与本地数据

默认情况下，Comfort Cues 的运行与 profile 管理应按本地工具处理：

- profile、绑定记录和应用状态保存在本机用户目录，例如 `%APPDATA%\Comfort Cues\`。
- profile 可能包含进程名片段、窗口标题片段和最后绑定的窗口信息。
- 不应自动上传窗口标题、进程名、日志或 profile。
- 如果未来加入更新检查、错误上报或云同步，必须在发布前更新隐私政策并提供用户选择。

用户请求支持时，建议只收集最小必要信息：

- Comfort Cues 版本。
- Windows 版本。
- 游戏名、窗口模式、是否使用反作弊。
- 是否使用 Steam/Discord/Xbox/NVIDIA/AMD 等覆盖层。
- 可选日志或截图；用户应先自行检查其中是否包含个人窗口标题。

## 健康与免责声明

Comfort Cues 不是医疗设备，不提供诊断、治疗或医学建议。它只是为部分用户提供额外视觉参考点的桌面辅助工具。

使用建议：

- 如果出现头晕、恶心、眼部疲劳、头痛或其他不适，请停止使用并休息。
- 调整游戏内 FOV、动态模糊、镜头晃动、帧率限制、鼠标灵敏度和显示亮度，通常也会影响舒适度。

## 常见问题

### 为什么独占全屏不可用？

独占全屏通常会绕过桌面合成器或改变窗口层级。Comfort Cues 是外部桌面覆盖层，不进入游戏渲染管线，因此不能保证覆盖在独占全屏游戏上方。

### 为什么有些游戏显示正常，有些不行？

不同游戏的窗口模式、反作弊、渲染后端、DPI 行为和覆盖层策略不同。Comfort Cues 优先支持普通窗口化和无边框窗口化场景。

### 为什么关闭主窗口后程序还在？

当前应用可驻留系统托盘。请从托盘菜单重新打开或退出。

### 杀毒软件提示风险怎么办？

先确认下载来源和 SHA-256 校验和。早期小众 Windows 工具可能出现误报，尤其是包含桌面覆盖层、输入读取或打包运行库时。不要要求用户关闭安全软件；应提供签名、校验和、发布说明和可复现构建信息来降低误报。

### 可以在比赛或排位中使用吗？

不作保证。用户必须遵守游戏、平台、服务器、赛事和反作弊规则。若规则不明确，建议不要在有账号风险或竞技约束的环境中使用。
